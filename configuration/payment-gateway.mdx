BeanQuick uses Mercado Pago to process payments for multi-vendor marketplace transactions. Mercado Pago provides a secure payment gateway supporting credit cards, debit cards, and various local payment methods across Latin America.

## Prerequisites

<Steps>
  <Step title="Create Mercado Pago Account">
    Sign up for a Mercado Pago account at [mercadopago.com](https://www.mercadopago.com)
  </Step>
  
  <Step title="Access Your Credentials">
    Navigate to [Developers Dashboard](https://www.mercadopago.com/developers/panel) to get your API credentials
  </Step>
  
  <Step title="Install Dependencies">
    The Mercado Pago SDK is already included in `composer.json`:
    ```bash
    composer install
    ```
  </Step>
</Steps>

## Mercado Pago SDK

BeanQuick uses the official Mercado Pago PHP SDK:

```json composer.json
"mercadopago/dx-php": "^3.8"
```

This package is automatically installed when you run `composer install`.

## Configuration

### Environment Variables

Add your Mercado Pago credentials to `.env`:

```bash .env
MERCADOPAGO_ACCESS_TOKEN=your-access-token-here
WEBHOOK_URL=https://yourdomain.com/api/webhook/mercadopago
APP_FRONTEND_URL=http://localhost:5173
```

| Variable | Description | Where to Find |
|----------|-------------|---------------|
| `MERCADOPAGO_ACCESS_TOKEN` | Private access token for API requests | Developer Dashboard > Credentials |
| `WEBHOOK_URL` | Public URL for payment notifications | Your backend domain + `/api/webhook/mercadopago` |
| `APP_FRONTEND_URL` | Frontend application URL for redirects | Your frontend domain |

<Warning>
The `WEBHOOK_URL` must be publicly accessible via HTTPS. Mercado Pago cannot send webhooks to `localhost`.
</Warning>

### Service Configuration

The access token is configured in `config/services.php`:

```php config/services.php
'mercadopago' => [
    'access_token' => env('MERCADOPAGO_ACCESS_TOKEN'),
],
```

## Getting API Credentials

### Sandbox Credentials (Testing)

1. Go to [Mercado Pago Developers](https://www.mercadopago.com/developers/panel)
2. Select **Test Credentials**
3. Copy the **Access Token**
4. Add to `.env` as `MERCADOPAGO_ACCESS_TOKEN`

### Production Credentials

1. Complete your Mercado Pago account verification
2. Go to [Mercado Pago Developers](https://www.mercadopago.com/developers/panel)
3. Select **Production Credentials**
4. Copy the **Access Token**
5. Update `.env` with production token

<Tip>
Use test credentials during development and only switch to production credentials when deploying to production.
</Tip>

## Webhook Configuration

Webhooks notify your application when payment status changes.

### Webhook URL Setup

The webhook endpoint is defined in `routes/api.php`:

```php
Route::post('/webhook/mercadopago', [PagoController::class, 'webhook']);
```

Full webhook URL format:
```
https://yourdomain.com/api/webhook/mercadopago
```

### Webhook Events

The webhook handler (`PagoController@webhook`) processes payment status updates:

- `approved` - Payment approved
- `pending` - Payment pending
- `in_process` - Payment being processed
- `rejected` - Payment rejected
- `cancelled` - Payment cancelled
- `refunded` - Payment refunded
- `charged_back` - Payment charged back

### Payment Status Mapping

BeanQuick maps Mercado Pago statuses to internal order states:

| Mercado Pago Status | BeanQuick Status | Action |
|---------------------|------------------|--------|
| `approved` | `aprobado` | Deduct stock, empty cart, change order status to "Pagado" |
| `pending`, `in_process` | `pendiente` | Keep order in pending state |
| `rejected`, `cancelled` | `rechazado` | Mark payment as failed |
| `refunded`, `charged_back` | `reembolsado` | Mark as refunded |

### Configuring Webhooks in Mercado Pago

<Steps>
  <Step title="Access Developer Dashboard">
    Go to [Mercado Pago Webhooks](https://www.mercadopago.com/developers/panel/webhooks)
  </Step>
  
  <Step title="Add New Webhook">
    Click "Add Webhook" or "Configure Webhooks"
  </Step>
  
  <Step title="Set Webhook URL">
    Enter your webhook URL:
    ```
    https://yourdomain.com/api/webhook/mercadopago
    ```
  </Step>
  
  <Step title="Select Events">
    Choose **Payment** events to receive notifications
  </Step>
  
  <Step title="Save Configuration">
    Save the webhook configuration
  </Step>
</Steps>

## Payment Flow

### 1. Create Payment Preference

When a user initiates payment, BeanQuick creates a payment preference:

```php
POST /api/pagos/{pedido_id}
```

This generates a Mercado Pago checkout URL (`init_point`) that redirects users to the payment page.

### 2. User Completes Payment

The user is redirected to Mercado Pago to complete the payment.

### 3. Payment Callback URLs

After payment, users are redirected based on status:

```php
"back_urls" => [
    "success" => env('APP_FRONTEND_URL') . "/pago-exitoso",
    "failure" => env('APP_FRONTEND_URL') . "/pago-fallido",
    "pending" => env('APP_FRONTEND_URL') . "/pago-pendiente"
]
```

### 4. Webhook Notification

Mercado Pago sends a webhook notification to update order status:

```
POST https://yourdomain.com/api/webhook/mercadopago
```

The webhook handler:
1. Retrieves payment details from Mercado Pago API
2. Updates order payment status
3. Deducts product stock (if approved)
4. Empties user's shopping cart (if approved)

### Local Webhook Testing with ngrok

Since webhooks require public URLs, use [ngrok](https://ngrok.com/) for local testing:

```bash

php artisan serve

# In another terminal, start ngrok
ngrok http 8000
```

```bash
WEBHOOK_URL=https://abc123.ngrok.io/api/webhook/mercadopago
```

Configure the ngrok webhook URL in the Mercado Pago developer dashboard for testing.
</Tip>

## Security Considerations

<Steps>
  <Step title="Validate Webhook Origin">
    Always verify webhook requests come from Mercado Pago by checking the payment status via API.
  </Step>
  
  <Step title="Use HTTPS in Production">
    Webhook URLs must use HTTPS in production for secure communication.
  </Step>
  
  <Step title="Protect Access Token">
    Never commit your access token to version control. Use environment variables.
  </Step>
  
  <Step title="Idempotency">
    The webhook handler checks existing payment status before updating to prevent duplicate processing.
  </Step>
</Steps>

## Troubleshooting

### Webhook Not Receiving Notifications

1. Verify webhook URL is publicly accessible:

```bash
curl -X POST https://yourdomain.com/api/webhook/mercadopago
```

2. Check Mercado Pago webhook logs in the developer dashboard

3. Review Laravel logs:

```bash
tail -f storage/logs/laravel.log
```

### Payment Status Not Updating

1. Check that `MERCADOPAGO_ACCESS_TOKEN` is correct

2. Verify the `external_reference` matches the order ID

3. Check database transaction logs

### Invalid Access Token

Clear configuration cache:

```bash
php artisan config:clear
php artisan cache:clear
```

### CORS Issues

If frontend cannot communicate with payment endpoints, verify CORS configuration in `config/cors.php`.
