## Overview

The Businesses API allows administrators to view, update, and delete business accounts in the marketplace. These endpoints manage already-activated businesses (not pending applications).

<Note>
All business management endpoints require:
- Valid Sanctum authentication token
- User must have `admin` role
</Note>

<Info>
For managing business **applications** (approval/rejection), see the [Business Applications API](/api/admin/applications).
</Info>

Updates business information.

### Endpoint

<ParamField path="PUT" type="string">
  /api/admin/empresas/{id}
</ParamField>

### Path Parameters

<ParamField path="id" type="integer" required>
  The ID of the business to update
</ParamField>

### Request Body

<ParamField body="nombre" type="string" required>
  Business name. Maximum 255 characters.
</ParamField>

<ParamField body="nit" type="string" required>
  Tax identification number. Must be unique (excluding current business).
</ParamField>

<ParamField body="telefono" type="string" required>
  Phone number
</ParamField>

<ParamField body="direccion" type="string" required>
  Business address
</ParamField>

### Validation Rules

- `nombre`: Required, string, max 255 characters
- `nit`: Required, string, unique (ignoring current business)
- `telefono`: Required, string
- `direccion`: Required, string

<Note>
The response includes the `usuario` relationship to ensure the frontend has access to the owner's email and other account details.
</Note>

### Response Fields

<ResponseField name="message" type="string">
  Success confirmation message
</ResponseField>

<ResponseField name="empresa" type="object">
  The updated business object with user relationship loaded
</ResponseField>

## Delete Business

<RequestExample>
```bash curl
curl -X DELETE https://api.beanquick.com/api/admin/empresas/1 \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Accept: application/json"
```
</RequestExample>

<ResponseExample>
```json
{
  "message": "Empresa eliminada correctamente."
}
```
</ResponseExample>

Permanently deletes a business account.

### Endpoint

<ParamField path="DELETE" type="string">
  /api/admin/empresas/{id}
</ParamField>

### Path Parameters

<ParamField path="id" type="integer" required>
  The ID of the business to delete
</ParamField>

### Response Fields

<ResponseField name="message" type="string">
  Confirmation message
</ResponseField>

### Cascade Deletion

<Warning>
Deleting a business may cascade delete associated products if configured with `onDelete('cascade')` in your database migrations. Verify your migration configuration before deleting businesses with active products.
</Warning>

### Error Responses

<ResponseExample>
```json Server Error
{
  "message": "Error al eliminar empresa."
}
```
</ResponseExample>

## Business Model

The `Empresa` model has the following structure:

### Database Fields

- `id` - Primary key (auto-increment)
- `user_id` - Foreign key to associated user account
- `nombre` - Business name
- `nit` - Tax identification number (unique)
- `direccion` - Business address
- `telefono` - Phone number
- `descripcion` - Business description
- `logo` - Logo file path in storage
- `foto_local` - Location photo file path in storage
- `is_open` - Boolean indicating if accepting orders
- `created_at` - Creation timestamp
- `updated_at` - Last update timestamp

### Auto-Appended Fields

The following fields are automatically added to JSON responses:

- `logo_url` - Full URL to logo (e.g., `https://api.beanquick.com/storage/logos/abc123.jpg`)
- `foto_local_url` - Full URL to location photo

### Relationships

- **usuario**: Belongs to one User (role: `empresa`)
- **productos**: Has many products
- **pedidos**: Has many orders

### Business Status

The `is_open` field controls whether the business is currently accepting orders:

- `true` - Business is open and accepting orders
- `false` - Business is closed (customers cannot place orders)

Business owners can toggle this status via the `/api/empresa/toggle-estado` endpoint.
