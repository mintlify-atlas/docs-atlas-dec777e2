title: "Quickstart"
description: "Get BeanQuick running locally in under 10 minutes"
---

Get your BeanQuick multi-vendor marketplace up and running locally in minutes. This guide will have you serving customers in no time.

<Steps>
  <Step title="Clone the repository">
    Clone the BeanQuick repository to your local machine:

    ```bash
    git clone <repository-url>
    cd beanquick
    ```
  </Step>

  <Step title="Install backend dependencies">
    Navigate to the backend directory and install PHP dependencies using Composer:

    ```bash
    cd back-end
    composer install
    ```

    <Note>
      BeanQuick requires **PHP 8.2 or higher** and **Composer** installed on your system.
    </Note>
  </Step>

  <Step title="Configure environment">
    Copy the example environment file and generate an application key:

    ```bash
    cp .env.example .env
    php artisan key:generate
    ```

    Update your `.env` file with your database credentials:

    ```bash .env
    DB_CONNECTION=mysql
    DB_HOST=127.0.0.1
    DB_PORT=3306
    DB_DATABASE=back_end
    DB_USERNAME=root
    DB_PASSWORD=your_password
    ```

    <Tip>
      Make sure your MySQL database server is running and the database `back_end` exists before proceeding.
    </Tip>
  </Step>

  <Step title="Run database migrations">
    Create the database tables and seed with initial data:

    ```bash
    php artisan migrate --seed
    ```

    This will create all necessary tables and set up an admin user with the following credentials:

    <Info>
      **Admin Credentials**
      - Email: `admin@gmail.com`
      - Password: `Admin12345`
    </Info>
  </Step>

  <Step title="Install frontend dependencies">
    Open a new terminal, navigate to the frontend directory, and install Node dependencies:

    ```bash
    cd front-end
    npm install
    ```

    <Note>
      BeanQuick frontend requires **Node.js 18 or higher** and **npm** installed.
    </Note>
  </Step>

  <Step title="Start the development servers">
    You have two options to start the servers:

    ### Option 1: Using the automated script (Recommended)

    From the `back-end` directory, run:

    ```bash
    composer run dev
    ```

    This single command starts:
    - Laravel development server (port 8000)
    - Queue worker for background jobs
    - Vite development server for hot module reloading

    ### Option 2: Manual startup

    Start each service in separate terminal windows:

    <CodeGroup>
      ```bash Backend Server
      cd back-end
      php artisan serve
      ```

      ```bash Queue Worker
      cd back-end
      php artisan queue:listen --tries=1
      ```

      ```bash Frontend Dev Server
      cd front-end
      npm run dev
      ```
    </CodeGroup>
  </Step>

  <Step title="Access the application">
    Open your browser and navigate to:

    - **Frontend**: [http://localhost:5173](http://localhost:5173)
    - **Backend API**: [http://localhost:8000](http://localhost:8000)

    <Info>
      Log in with the admin credentials (`admin@gmail.com` / `Admin12345`) to access the admin panel and start configuring your marketplace.
    </Info>
  </Step>
</Steps>

## Next Steps

Now that BeanQuick is running:

- Configure your marketplace settings
- Add product categories
- Set up payment gateways (MercadoPago is pre-integrated)
- Invite vendors to join your platform
- Customize your storefront theme

<Warning>
  The default admin credentials should be changed immediately in production environments. Update them through the admin panel or user settings.
</Warning>
